# 第一章 三种基础的数据结构

# 栈

这种存取方式的特点可总结为先进先出，后进后出（ LIFO，Last In，First Out ）。

# 栈在不同应用场景的含义

  1. 栈是一种数据结构，它表达的是数据的一种存取方式，是一种理论基础。
  2. 栈可用来规定代码的执行顺序，在 JavaScript 中叫作函数调用栈（ call stack ）。
  3. 栈表达的是一种数据在内存中的存储区域，通常叫做栈区。但是 JavaScript 作为一门高级语言，并没有同其他语言那样区分栈区或堆区，因此在这里不做扩展。我们可以简单粗暴地认为在 JavaScript 中，所有地数据都是存放在堆内存空间中地。

# 堆

堆数据结构通常是一种树状结构。

下图是 testHeap 示意图

该示意图可以用字面量对象的形式体现出来：

const testHeap = {
  a: 10,
  b: 20,
  c: {
    m: 100,
    n: 110
  }
}

当我们想要访问 a 时，只需通过testHeap.a来访问即可，而不用关心 a、b、c 的具体顺序。

# 队列

在 JavaScript 中，理解队列数据结构的目的是为了搞清楚时间循环（ Event Loop ）机制到底是怎么回事。

队列（ queue ）是一种先进先出（ FIFO ）的数据结构。正如排队过安检一样，排在队伍前面的人一定是最先过安检的人。
